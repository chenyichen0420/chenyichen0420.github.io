思路分析

简单题。

首先，我们注意到 $k$ 非常大，猜测 $k$ 的具体数值应该关系不大。

于是我们想到了经典套路：对称构造。说白了，就是如果有两个完全一样的博弈对象，那么我总是可以通过在另一个对象上的等价操作来抵消你在其中一个对象上的操作。

于是我们不难看出，操作效果只和 $k$ 的奇偶性有关。如果 $k$ 是偶数，那么相当于将整个堆直接删掉，如果 $k$ 是奇数，那么相当于将整个堆砍一半。

于是我们分情况讨论 SG 函数的数值。

如果 $k$ 是偶数，显然所有的偶数的 SG 函数值都应该是非零的，那么对应的有所有 $>1$ 的奇数的 SG 函数值是 $0$，特例就是 $n=1$ 时 SG 为 $1$，$n=2$ 时 SG 为 $2$。

如果 $k$ 是奇数，我们打一个表，注意到 $n=5$ 的时候 SG 值为 $0$，那么显然 $n=6$ 时 SG 不为 $0$，则 $n=7$ 时 SG 为 $0$，以此类推。

因此 $n\ge 5$ 的奇数的 SG 函数值一定是 $0$，偶数呢？直接按照定义暴力计算就行了，反正每一次都砍半，时间复杂度只有 $\log$。

此题可解，参考代码如下：

CPP 
#include<bits/stdc++.h>
using namespace std;
#define int long long
constexpr int mod=1000000007;
int n,k,a[100005],sg;
inline int sg0(int v){
	return (v<=2)?v:1-(v&1);
}
inline int sg1(int v){
	if(v<=3) return v&1;
	if(v==4) return 2;
	if(v&1) return 0;
	return 1+(sg1(v/2)==1);
}
signed main() {
	ios::sync_with_stdio(0); cin>>n>>k;
	for(int i=1;i<=n;++i) cin>>a[i];
	/*
	如果 k 是偶数，那么相当于直接把这堆拿空
	如果 k 是奇数，那么相当于把这堆除半
	上面的结论显然可以通过对称构造直接得出
	于是分情况讨论一下答案

	如果 k 是偶数：
	sg(0~inf) = [0,1,2,0,1,0,1,0,1,...]
	总之 sg<=2 的时候特殊处理一下，剩下的奇数为 0，偶数为 1。

	如果 k 是奇数：
	sg(0~inf) = [0,1,0,1,2,0,1,0,1,...]
	             0 1 2 3 4 5 6 7 8
	直接递归计算是不是就行了，反正奇数都是 0
	*/
	if(k&1) for(int i=1;i<=n;++i) sg^=sg1(a[i]);
	else for(int i=1;i<=n;++i) sg^=sg0(a[i]);
	puts(sg?"Kevin":"Nicky");
}

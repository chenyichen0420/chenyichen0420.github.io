思路分析

重点在于图论的转化，既是简单的地方也是有意思的地方。

首先当我们看到我们需要以这种神奇的方式拼拼图时，我们不难想到对于悬空的需要可行的支撑物，对于不悬空的可以用来支撑。

其实如果我们允许拼图水平翻转，就类似于在对于这些长度找欧拉路径。

可是他不让我们反转！只能右边的凸补左边的凹，左边的凸补右边的凹。

这就是这道题有意思的地方。既然左右凸，左右凹不是同一类，那我们就直接不把它当成同一类存储就完了。

对于一块拼图，当 $c=0$，也就是左凸时，我们视为他有一个 $-a$ 的凸，反之视为他有一个 $c$ 的凹。当 $d=0$，也就是右凸时，我们视为他有一个 $b$ 的凸，反之视为他有一个 $-d$ 的凹。

这样，我们用正负分类的方式巧妙而简便的将左右的凹凸区分开了。为了不使用 map，我们在给它加上一个 $h$ 转为正数方便处理。

然后就是去构造若干条不相交的路径，满足从正点出发，负点终止。且每条边都被覆盖（一次）。

可以通过度数判断一个连通块是否可以构造出这样的一堆路径。我们直接暴搜这样的路径存不存在，对于每一坨连通块，如果其中有出入度不一的点那就是可行的。反之则不可行。

另外还有一点：我们可以视为左端对应的点度数减一，右端对应的点度数加一，然后判断是不是所有左部点度数 $\le0$，右端度数 $\ge0$。因为我们是从左向右拼，因而左端作为入边度数恒负，右端作为出边度数恒正。

代码如下：

CPP 
#include<bits/stdc++.h>
using namespace std;
vector<int>son[405]; bool vis[405], cn;
int ds[405], n, h, a, b, c, d, l, r;
inline void dfs(int p) {
	vis[p] = 1; cn |= ds[p];
	for (int sp : son[p]) if (!vis[sp]) dfs(sp);
}
signed main() {
	ios::sync_with_stdio(0); cin >> n >> h; vis[h] = 1;
	for (int i = 1;i <= n;++i)
		cin >> a >> b >> c >> d,
		l = c ? c + h : h - a,
		r = d ? h - d : b + h,
		--ds[l], son[l].emplace_back(r),
		++ds[r], son[r].emplace_back(l);
	//用正负区分左右部的边，加 h 保证正数。
	for (int i = 0;i ^ h;++i)
		if (ds[i] > 0) return puts("NO"), 0;
	//对于左部的应保证入度非正
	for (int i = h << 1;i ^ h;--i)
		if (ds[i] < 0) return puts("NO"), 0;
	//对于右部的应保证入度非负
	for (int i = 0;i <= h << 1;++i)
		if (!vis[i] && son[i].size()) //注意点都没有的不被考虑
			if (cn = 0, dfs(i), !cn) return puts("NO"), 0;
	puts("YES");
}

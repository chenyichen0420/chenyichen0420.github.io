暴力模拟的问题在于我们可以一次性插入 109 个数，删除 109 个数，两次操作就炸了。

但是插入的数又非常美妙的性质：连续且相等的一堆数。这启示我们直接维护“数段”而不是数本身。

具体的，插入时我们记录两个值：数及其个数。删除的时候，我们不停的取最开头的数段，统计贡献并直接抛弃，直到剩余可取的数的个数比开头的数段的数的个数少。我们统计剩余贡献后删除相应个数的数即可。

代码如下：

#include <bits/stdc++.h>
using namespace std;
#define int long long
int n, ans; queue<pair<int, int>>v;
signed main(){
	ios::sync_with_stdio(0);
	cin >> n;
	for (int i = 1, o, a, b; i <= n; ++i)
		if (cin >> o >> a, o & 1)
			cin >> b, v.emplace(make_pair(b, a));
		else {
			while (a > v.front().second)
				ans += v.front().first * v.front().second,
				a -= v.front().second, v.pop();
			ans += v.front().first * a;
			v.front().second -= a;
			cout << ans << endl; ans = 0;
		}
}

首先不考虑修改，我们发现一个合法的序列在排序后必然满足 ai=ai+1a_i=a_{i+1}ai​=ai+1​ 或者 ai=ia_i=iai​=i，并且二者不同时满足。

也就是说，这些数会形成许多右端点在 ai=ia_i=iai​=i 的直线上且对应区间并集恰为 [1,n][1,n][1,n] 的许多平台。

于是正解就出来了。

对于每一个数，我们统记他出现的次数。iii 出现了 pip_ipi​ 次就会对应一个覆盖 (i−pi,i](i-p_i,i](i−pi​,i] 的区间。最后再来看一下哪些点没被覆盖就完了。

你说这是 O(nm)O(nm)O(nm) 的，我说每次只会动两个区间共 222 个位置，算上预处理时间复杂度是 O(n+m)O(n+m)O(n+m) 的。

代码如下：

CPP 
#include<bits/stdc++.h>
using namespace std;
int n, m, a[200005], t[200005], c[200005], ans;
inline void add(int p) {
	int cp = p - t[p]; t[p]++;
	if (cp > 0 && !c[cp]++) ans--;
}
inline void del(int p) { 
	t[p]--; int cp = p - t[p];
	if (cp > 0 && !--c[cp]) ans++;
}
signed main() {
	ios::sync_with_stdio(0); cin >> n >> m; ans = n;
	for (int i = 1;i <= n;++i) cin >> a[i], add(a[i]);
	for (int i = 1, x, y;i <= m;++i)
		cin >> x, del(a[x]), cin >> a[x], add(a[x]), cout << ans << endl;
}

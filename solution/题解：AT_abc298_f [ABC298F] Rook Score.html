思路分析

显然选择 (i,j) 的贡献为 sxi​+syj​−vi,j​。其中 sxi​=∑vi,y​,syj​=∑vx,j​。

于是我们不难想到一个思路：枚举 i，快速找出 syj​−vi,j​ 最小的 j。

怎么样快速找呢？我们注意到实际上会被 vi,j​ 影响的 (i,j) 只会有 n 个，也就是说相对而言变化是比较少的。

于是我们可以使用 multiset 来维护可行的值的集合 ，并使用 unordered_map<int,pair<int,int>> 记录每一个 i 位置对应的有变化的 (i,j)，外加一个 unordered_map 记录对应 y 轴的值。

纯纯的 STL 常数套装，不过能过。代码如下：

#include<bits/stdc++.h>
using namespace std;
#define int long long
int n, a, b, c, ans; unordered_map<int, int>sx, sy;
map<int, vector<pair<int, int>>>ap; multiset<int>v;
signed main() {
	ios::sync_with_stdio(0); cin >> n;
	for (int i = 1; i <= n; ++i)
		cin >> a >> b >> c, sx[a] += c, sy[b] += c,
		ap[a].emplace_back(make_pair(b, c));
	for (const auto& y : sy) v.emplace(y.second);
	for (const auto& x : ap) {
		for (const auto& s : x.second) {
			v.erase(v.find(sy[s.first]));
			sy[s.first] -= s.second;
			v.emplace(sy[s.first]);
		}
		ans = max(ans, sx[x.first] + (*v.rbegin()));
		for (const auto& s : x.second) {
			v.erase(v.find(sy[s.first]));
			sy[s.first] += s.second;
			v.emplace(sy[s.first]);
		}
	}
	cout << ans << endl;
}

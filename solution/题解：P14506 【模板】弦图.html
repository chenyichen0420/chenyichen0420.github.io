基本概念与定义

约定原图 G=(V,E)G=(V,E)G=(V,E)。

定义完全图为 ∀u,v∈V,u≠v,(u,v)∈E\forall u,v\in V,u\neq v,(u,v)\in E∀u,v∈V,u=v,(u,v)∈E 的图。

定义诱导子图（导出子图）为 G0=(V0,E0),V0⊆V,E0={(u,v)∣u,v∈V0,(u,v)∈E}G_0=(V_0,E_0),V_0\subseteq V,E_0=\{(u,v)\mid u,v\in V_0,(u,v)\in E\}G0​=(V0​,E0​),V0​⊆V,E0​={(u,v)∣u,v∈V0​,(u,v)∈E}。

定义团为原图的一个导出子图，满足其为完全图。

定义极大团为不是其他团的子集的团。

定义最大团为点数最多的团。其中一个图的团数定义为最大团的点数，简记为 ω(G)\omega(G)ω(G)。

定义最小染色为用最少的颜色给点染色，使相邻点颜色不同，答案称为色数，简记为 χ(G)\chi(G)χ(G)。

定义最大独立集为最大的一个点的子集，使得任何两个点不相邻，集合大小简记为 α(G)\alpha(G)α(G)。

定义最小团覆盖为用最少的团覆盖所有的点，其团数定义为 κ(G)\kappa(G)κ(G)。

对于任意图，有如下性质：

  1. ω(G)≤χ(G)\omega(G)\le\chi(G)ω(G)≤χ(G)；
  2. α(G)≤κ(G)\alpha(G)\le\kappa(G)α(G)≤κ(G)。

定义弦为一个简单环中连接不相邻的两个点的边。

定义弦图为图中任意长度大于 333 的简单环都至少有一个弦的图。左图是弦图，右图不是。

对于弦图，有如下性质：

  1. 弦图的每一个诱导子图一定是弦图；
  2. 弦图的任一个诱导子图不同构于 Cn (n>3)C_n\ (n>3)Cn​ (n>3)，图论中 CnC_nCn​ 为 nnn 点环。

此外，给定一些区间，定义一个相交图为每个顶点表示一个区间，两个点有边当且仅当两个区间的交集非空。一个图为区间图当它是若干个区间的相交图。我们还有性质：区间图一定是弦图。

完美消除序列与判定

令 N(v)\Nu(v)N(v) 表示与 vvv 相邻的点集，称一个点为单纯点当且仅当 V={v}∪N(v)V=\{v\}\cup\Nu(v)V={v}∪N(v) 的诱导子图为一个团。

有引理：任何一个弦图都至少有一个单纯点，不是完全图的弦图至少有两个不相邻的单纯点。

定义完美消除序列为一个序列 v1,v2,…,vnv_1,v_2,\dots,v_nv1​,v2​,…,vn​ 满足 ∀i∈[1,n]\forall i\in[1,n]∀i∈[1,n]，在 V={vi,vi+1,…,vn}V=\{v_i,v_{i+1},\dots,v_n\}V={vi​,vi+1​,…,vn​} 的诱导子图中 viv_ivi​ 是一个单纯点。

有定理：一个无向图是弦图当且仅当它有一个完美消除序列。证明：

充分性：由引理知任何一个弦图都至少有一个单纯点以及弦图的诱导子图都是弦图。可以使用数学归纳法假设当点数 <n<n<n 的弦图一定有完美消除序列，那么点数为 nnn 的弦图的完美消除序列可以由一个单纯点加上剩余点的诱导子图的完美消除序列得到。

必要性：反证若无向图存在一个长度 >3>3>3 的无弦环，不妨设环中在完美消除序列中出现最前面的点为 vvv，设环中 vvv 与 v1,v2v_1,v_2v1​,v2​ 相连，根据完美消除序列的性质知 v1,v2v_1,v_2v1​,v2​ 相连，与环无弦矛盾。所以无向图为弦图。

对弦图求完美消除序列有两个 O(m+n)O(m+n)O(m+n) 的算法，分别是字典序广度优先搜索（LexBFS）和最大势算法（MCS），下面介绍后者：

从 nnn 到 111 的顺序依次给点标号（标号为 iii 的点出现在完美消除序列的第 iii 个）。

设 labelilabel_ilabeli​ 表示第 iii 个点与多少个已标号的点相邻，每次选择 labelilabel_ilabeli​ 最大的未标号的点进行标号。

我们将这些点挂到其 labellabellabel 对应的链表上进行维护即可，时间复杂度是 O(n+m)O(n+m)O(n+m) 的。

完美消除序列的判定：

设 {vi+1,…,vn}\{v_{i+1},\dots,v_n\}{vi+1​,…,vn​} 中所有与 viv_ivi​ 相邻的点依次为 vj1,…,vjkv_{j_1},\dots,v_{j_k}vj1​​,…,vjk​​。只需判断 vj1v_{j_1}vj1​​ 是否与 vj2,…,vjkv_{j_2},\dots,v_{j_k}vj2​​,…,vjk​​ 相邻即可。时间复杂度为 O(n+m)O(n+m)O(n+m)。

重点在这里！很多人这一步没有写线性的，不过线性的其实很好写。

好像常见的做法是带个 log⁡\loglog 或者用哈希表的，有一种严格 O(n+m)O(n+m)O(n+m) 的方法：我们直接把询问拍到点上。

如果一些点共享了同一个 vj1v_{j_1}vj1​​，那么可以直接把他们一并处理，只需要将他们挂到 vj1v_{j_1}vj1​​ 的询问下面即可。显然，这个操作的常数是不大的，并且严格线性。

判定就简单了：如果图为弦图，那么最大势算法会给出一个合法的完美消除序列，反之则不会，因此跑一遍最大势再跑一遍判定即可，时间复杂度 O(n+m)O(n+m)O(n+m)。

极大团

设第 iii 个点在弦图的完美消除序列第 pip_ipi​ 个。令 N(v)={w∣w与v相邻且pw>pv}\Nu(v)=\{w\mid w与v相邻且p_w>p_v\}N(v)={w∣w与v相邻且pw​>pv​}，则弦图的极大团一定是 {v}∪N(v)\{v\}\cup\Nu(v){v}∪N(v) 的形式。证明：

设点集 VVV 的诱导子图为弦图的极大团，设 vvv 为 VVV 中 ppp 值最小的点，即出现在完美消除序列中最前面的点。由于 V⊆{v}∪N(v)V\subseteq\{v\}\cup\Nu(v)V⊆{v}∪N(v) 为一个团，VVV 为极大团，所以 V={v}∪N(v)V=\{v\}\cup\Nu(v)V={v}∪N(v)。

推论：弦图最多有 nnn 个极大团。显然只需要检验每一个 {v}∪N(v)\{v\}\cup\Nu(v){v}∪N(v) 是不是极大团即可。

如何判断 A={v}∪N(v)A=\{v\}\cup\Nu(v)A={v}∪N(v) 是不是极大团：若存在 B={w}∪N(w)B=\{w\}\cup\Nu(w)B={w}∪N(w) 使得 A⊆BA\subseteq BA⊆B，则 AAA 不是极大团。

色数与团数

我们按照完美消除序列从后往前依次给每个点染色，给每个点染上可以染的最小的颜色即可。证明：

设使用了 TTT 种颜色，则 T≥χ(G)T\ge\chi(G)T≥χ(G)，又因为 T=κ(G)≤χ(G)T=\kappa(G)\le\chi(G)T=κ(G)≤χ(G)，所以 κ(G)=χ(G)=T\kappa(G)=\chi(G)=Tκ(G)=χ(G)=T。

最大独立集与最小团覆盖

最大独立集：完美消除序列从前往后能选就选。

最小团覆盖：设最大独立集为 {p1,p2,…,pt}\{p_1,p_2,\dots,p_t\}{p1​,p2​,…,pt​}，则 {p1∪N(p1),…,pt∪N(pt)}\{p_1\cup\Nu(p_1),\dots,p_t\cup\Nu(p_t)\}{p1​∪N(p1​),…,pt​∪N(pt​)} 为最小团覆盖。证明：

由独立集，t≤α(G)t\le\alpha(G)t≤α(G)；由团覆盖，t≥κ(G)t\ge\kappa(G)t≥κ(G)；结合 α(G)≤κ(G)\alpha(G)\le\kappa(G)α(G)≤κ(G) 得 α(G)=κ(G)=t\alpha(G)=\kappa(G)=tα(G)=κ(G)=t，即最大独立集数等于最小团覆盖数。

#include<iostream>
#include<string>
#include<algorithm>
#include<numeric>
#include<sstream>
#include<map>
#include<vector>
#include<set>
#include<cstring>
#include<queue>
#include<cassert>
#include<cmath>
using namespace std;
//#define int long long
int t, n, m, lab[100005], rk[100005], ord[100005], tag[100005];
vector<int>son[100005], va[100005], q[100005]; bool vis[100005];
signed main() {
	ios::sync_with_stdio(0);
	for (cin >> t;t;t--) {
		cin >> n >> m; va[0].clear();
		for (int i = 1;i <= n;++i)
			son[i].clear(), va[i].clear(), q[i].clear();
		for (int i = 1;i <= n;++i)
			vis[i] = lab[i] = rk[i] = tag[i] = 0;
		for (int i = 1, l, r; i <= m; ++i)
			cin >> l >> r,
			son[l].emplace_back(r),
			son[r].emplace_back(l);
		/*
		MCS 给出候选完美消除序列：
		从 $n$ 到 $1$ 的顺序依次给点标号(标号为 $i$ 的点出现在完美消除序列的第 $i$ 个)。
		设 $label_i$ 表示第 $i$ 个点与多少个已标号的点相邻，
		每次选择 $label_i$ 最大的未标号的点进行标号。
		我们将这些点挂到其 $label$ 对应的表上进行维护就行了，时间复杂度是 $O(n+m)$ 的。
		检验完美消除序列是否合法：
		设 $\{v_{i+1},\dots,v_n\}$中所有与 $v_i$ 相邻的点依次为 $v_{j_1},\dots,v_{j_k}$。
		只需判断 $v_{j_1}$ 是否与 $v_{j_2},\dots,v_{j_k}$ 相邻即可。时间复杂度为 $O(n+m)$。
		好像常见的做法是带个 log 或者用 hash 表的，有一种严格 $O(n+m)$ 的方法：
		我们直接把询问拍到点上：如果一些点共享了同一个 $v_{j_1}$，
		那么可以直接把他们一并处理，只需要将他们挂到 $v_{j_1}$ 的询问下面就行了。
		显然，这个操作的常数是不大的。
		*/
		int mxp = 0, cnt = n, o = 0, a = 0; bool cn = 1;
		for (int i = 1; i <= n; ++i) va[0].emplace_back(i);
		while (cnt >= 1) {
			while (mxp >= 0 && va[mxp].empty()) --mxp;
			int p = va[mxp].back();
			va[mxp].pop_back();
			if (vis[p]) continue;
			vis[p] = 1; rk[cnt--] = p;
			for (int sp : son[p]) if (!vis[sp]) {
				va[++lab[sp]].emplace_back(sp);
				if (lab[sp] > mxp) mxp = lab[sp];
			}
		}
		for (int i = 1; i <= n; ++i) ord[rk[i]] = i;
		for (int i = 1; i <= n; ++i) {
			int p = rk[i], mp = -1, mrk = n + 1, cnt = 0;
			for (int sp : son[p])
				if (ord[sp] > i) {
					++cnt;
					if (ord[sp] < mrk)
						mrk = ord[sp],
						mp = sp;
				}
			o = max(o, cnt + 1);
			if (mp != -1) q[mp].emplace_back(p);
			/*
			{
				for (int sp : son[mp]) tag[sp] = tva;
				for (int sp : son[p])
					if (ord[sp] > i && sp != mp)
						if (tag[sp] != tva) {
							cn = false;
							break;
						}
				if (!cn) break;
			}
			*/
		}
		/*
		上面顺求了一个 omega/chi(团数=色数)，流程如下：
		我们按照完美消除序列从后往前依次给每个点染色，给每个点染上可以染的最小的颜色即可：
		设使用了T种颜色, 则 $T\ge\chi(G)$，又因为 $T=\kappa(G)\le\chi(G)$，
		所以 $\kappa(G)=\chi(G)=T$。
		还没有检验完，离线后的检验流程在下面：
		*/
		for (int i = 1, p;i <= n && cn;++i) {
			if (!q[p = rk[i]].size()) continue;
			for (int sp : son[p]) tag[sp] = p;
			for (int j : q[p])
				for (int sp : son[j])
					if (ord[sp] > i && sp != p)
						if (tag[sp] != p) {
							cn = 0; break;
						}
		}
		if (!cn) {
			cout << "No\n";
			continue;
		}
		for (int i = 1;i <= n;++i) vis[i] = 0;
		/*
		随后我们考虑求解最大独立集大小（实际上也是最小团覆盖数）：
		最大独立集：完美消除序列从前往后能选就选。
		最小团覆盖：设最大独立集为 $\{p_1,p_2,\dots,p_t\}$，
		则 $\{p_1\cup\Nu(p_1),\dots,p_t\cup\Nu(p_t)\}$ 为最小团覆盖。证明：
		由独立集，$t\le\alpha(G)$；由团覆盖，$t\ge\kappa(G)$；
		结合 $\alpha(G)\le\kappa(G)$ 得 $\alpha(G)=\kappa(G)=t$，
		即最大独立集数=最小团覆盖数。
		*/
		for (int i = 1, p; i <= n; ++i)
			if (!vis[p = rk[i]]) {
				++a; for (int sp : son[p]) vis[sp] = 1;
			}
		cout << "Yes\n";
		for (int i = 1; i <= n; ++i) cout << rk[i] << " ";
		cout << endl << o << ' ' << o << ' ' << a << endl;
	}
	return 0;
}

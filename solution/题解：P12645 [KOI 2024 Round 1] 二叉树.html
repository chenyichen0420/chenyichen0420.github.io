思路分析

大约是个下位蓝吧。我们考虑从子树合并答案。

答案分成两部分：子树内的与跨子树的。

显然子树内的我们可以直接沿用已经求出来的答案。重点考虑跨子树的。

跨子树的一定左端点在左子树，右端点在右子树（废话），但是确实是一种常用的技巧：左右分别计算。

一个跨树的区间分成左子树区间和右子树区间。其中左子树区间是一个后缀区间，右子树是一个前缀区间。

对于一个后缀区间，假设需要 $a$ 次能够覆盖掉，那么对于最终的子树的贡献就是 $a$ 乘上右子树的叶子节点数。

前缀区间是类似的。特殊情况是左右区间均取完，实际上只需要取根节点就行了，注意特殊处理一下。

那么我们写一下转移。令 $s_i,ls_i,rs_i,ans_i$ 分别表示表示 $T_i$ 的叶子节点个数，前缀区间覆盖所需节点个数和，后缀区间覆盖所需节点个数和，所有区间覆盖所需节点个数和。

  • $s_i=s_l+s_r$。
  • $ls_i=ls_l+rs_r+s_l-1$。
  • $rs_i=rs_r+rs_l+s_r-1$。
  • $ans_i=ans_l+ans_r+rs_l\times s_r+ls_r\times s_l-1$。

实现出来即可。代码如下：

CPP 
#include<bits/stdc++.h>
using namespace std;
#define int long long
constexpr int mod=1e9+7;
int n,l,r,s[100005],a[100005],f[100005],b[100005];
signed main(){
	ios::sync_with_stdio(0);
	a[0]=s[0]=f[0]=b[0]=1; cin>>n;
	for(int i=1;i<=n;++i){
		cin>>l>>r;
		a[i]=a[l]+a[r]+f[l]*s[r]+b[r]*s[l]-1;
		a[i]%=mod;
		cout<<a[i]<<endl;
		s[i]=s[l]+s[r];
		f[i]=f[l]+f[r]+s[l]-1;
		b[i]=b[l]+b[r]+s[r]-1;
		s[i]%=mod;
		f[i]%=mod;
		b[i]%=mod;
	}
}

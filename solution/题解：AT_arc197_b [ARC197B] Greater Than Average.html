思路分析

一个相当复杂的思考方式。

首先，因为选的是子序列，所以说其实选出来的数在原数列的位置并不重要。因此，将他们排序不会影响结果。

然后我们思考一下，一个序列可以被分成两部分，一部分在平均值以上，一部分在平均值及以下。

我们不难发现，这两部分一定分别是连续的。因为更大的那部分可以将不连续的部分强制拍下去，使得总和更小，一定不劣，更小的部分同理。

然后我们假设更小的那部分的和是 sss，长度为 aaa，更大的那部分和是 ttt，长度为 bbb，最小的是 vvv。那不难发现 v>s+ta+bv>\frac{s+t}{a+b}v>a+bs+t​。

展开并移项可得 v×a−s>t−v×bv\times a-s>t-v\times bv×a−s>t−v×b。

然后我们知道在 vvv 前边的值比 vvv 小，所以说前面的全部取走一定是最优的。

我们知道在 vvv 后边的值比 vvv 大，所以说右式的值单调递增。

因此我们可以二分 bbb 的长度。代码如下：

CPP 
#include<bits/stdc++.h>
using namespace std;
#define int long long
int t, n, a[200005], sm[200005], ans;
signed main() {
	ios::sync_with_stdio(0);
	for (cin >> t;t;t--) {
		cin >> n; ans = 0;
		for (int i = 1;i <= n;++i) cin >> a[i];
		sort(a + 1, a + n + 1);
		for (int i = 1;i <= n;++i) sm[i] = sm[i - 1] + a[i];
		for (int i = 2;i <= n;++i) {
			int l = i - 1, r = n, mid;
			int vas = a[i] * (i - 1) - sm[i - 1];
			while (l != r) {
				mid = l + r + 1 >> 1;
				int tvb = (sm[mid] - sm[i - 1]) - a[i] * (mid - i + 1);
				if (tvb < vas) l = mid;
				else r = mid - 1;
			}
			ans = max(ans, l - i + 1);
		}
		cout << ans << endl;
	}
}

所以，你可以把上面那一大堆话理解为：最优解只出现在前缀当中。所以你可以用双指针 O(n)O(n)O(n) 切掉。

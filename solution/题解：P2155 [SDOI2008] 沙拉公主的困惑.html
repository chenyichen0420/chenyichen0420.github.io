显然答案是 n!m!×φ(m!)=n!×φ(m)m!\frac{n!}{m!}\times\varphi(m!)=n!\times\frac{\varphi(m)}{m!}m!n!​×φ(m!)=n!×m!φ(m)​。

所以我们可以分别预处理出 n!n!n! 和 φ(m)m!\frac{\varphi(m)}{m!}m!φ(m)​。

但是有一点小问题需要修理一下：有可能 n!n!n! 含有质因子 RRR，而且 m!φ(m)\frac{m!}{\varphi(m)}φ(m)m!​ 也有。最后乘起来正好抵消掉了。比如 1 3 4 3。

所以说在枚举的时候，我们可以特殊处理一下 i≡0(modR)i\equiv 0\pmod Ri≡0(modR) 的情况。

显然如果 i=Ri=Ri=R，那么我们令 φ(i)i!=φ(i−1)(i−1)!×(i−1)\frac{\varphi(i)}{i!}=\frac{\varphi(i-1)}{(i-1)!}\times(i-1)i!φ(i)​=(i−1)!φ(i−1)​×(i−1) 而不是 i−1i\frac{i-1}{i}ii−1​。相当于我们提前削掉了 m!φ(m)\frac{m!}{\varphi(m)}φ(m)m!​ 的一个 RRR。显然他最多也就只会有一个 RRR。

同样的，我们需要在同一个消掉 n!n!n! 的一个 RRR。而显然 n!n!n! 在 n=Rn=Rn=R 的时候正好有第一个 RRR。我们在这个地方让 n!=(n−1)!n!=(n-1)!n!=(n−1)! 即可。

那我们怎么知道什么时候应该有 RRR，使得答案为 000，什么时候没有呢？显然当且仅当 nnn 中 RRR 的因子数比 mmm 中多时才是 000。而显然一个极为简单的判断方式是 ⌊nR⌋>⌊mR⌋\lfloor\frac{n}{R}\rfloor>\lfloor\frac{m}{R}\rfloor⌊Rn​⌋>⌊Rm​⌋。

#include<bits/stdc++.h>
using namespace std;
constexpr bool online = 0;
#define int long long
int t, mod, n, m, p[1000005], pc[10000005], cnt, ans, jc[10000005]; 
bool isp[10000005];
inline int qpow(int a, int b, int p) {
	int ret = 1;
	for (; b; b >>= 1, a = a * a % p)
		(b & 1) && (ret = ret * a % p);
	return ret;
}
inline void tmod(int& l, const int r) { l = (l % mod + mod) % mod; }
inline int calc(int n, int m) {
	if (n / mod > m / mod) return 0;
	return jc[n] * pc[m] % mod;
}
signed main() {
	if (online)
		freopen("b.in", "r", stdin),
		freopen("b.out", "w", stdout);
	ios::sync_with_stdio(0); cin >> t >> mod; isp[1] = 1;
	for (int i = 2; i <= 1e7; ++i) {
		if (!isp[i]) p[++cnt] = i;
		for (int j = 1; j <= cnt && i * p[j] <= 1e7; ++j)
			if (isp[i * p[j]] = 1, i% p[j] == 0) break;
	}
	for (int i = jc[0] = pc[0] = 1; i <= 1e7; ++i)
		if (i != mod) {
			if (!isp[i])
				pc[i] = pc[i - 1] * qpow(i, mod - 2, mod) % mod * (i - 1) % mod;
			else pc[i] = pc[i - 1]; jc[i] = jc[i - 1] * i % mod;
		}
		else
			pc[i] = pc[i - 1] * (i - 1) % mod,
			jc[i] = jc[i - 1] * i / mod % mod;
	while (t--) cin >> n >> m, cout << calc(n, m) << endl;
}

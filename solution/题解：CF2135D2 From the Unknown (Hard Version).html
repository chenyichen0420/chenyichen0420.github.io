思路分析

D1，D2 题解二合一。

我们首先发现一件事情：如果我能够将答案锁定在一个区间 [l,r]，满足 l×2>r，那么我可以构造序列 l 1 l 2 l 3 l 4…l r-l+1，并通过结果直接判断出 w 是 [l,r] 当中的具体哪一个。

现在的问题就是怎么去快速锁定一个答案区间了。

对于弱化版问题，我们可以选择先询问 105 个 1。此时的答案区间就是 ⌈w105​⌉ 的值相等的那些区间。我们可以将这些区间全部打表出来，发现确实所有区间都是满足要求的。最长的区间是 [50000,99999]，仍然在长度限制内。

对于强化版问题，我们选择调整块长。如果我们把第一步的数值从 1 调整为 a，那么它本质上就会在 [1,a−1] 区间视为无解，并把剩下的部分每 a 个分一个大的块，然后区分出答案大致在那些大块里面。

这个东西的话，我们可以碰碰运气，或者写程序分析上界。总之我们发现，当我们取 a=128 时他是可行的。

具体的来说，为了区分 [1,127] 内的答案，你会需要在第二次输出 127×128 个 1。你还剩下 8744 次询问可用。

我们在第一次输出 8744 个 128，如果答案落在后面的区间，我们仍然套用上面的区分方式。不难打表证明这样做，区间满足二倍限制。其中最大的区间是 [86144,93311]，需要的询问次数不足 1.6×104，足够通过本题。

加强版代码如下：

#include<bits/stdc++.h>
using namespace std;
#define int long long
int t, v, l[1119235], r[1119235], v2, lm; set<int>s;
signed main() {
	ios::sync_with_stdio(0);
	for (int i = 128; i <= 1e5; ++i) {
		int t = ceil(8744.0 / (i / 128));
		if (!l[t]) l[t] = i; r[t] = i;
		s.emplace(t);
	}
	for (int i : s) cerr << l[i] << " " << r[i] << endl;
	for (cin >> t; t; t--) {
		cout << "? 8744 ";
		for (int i = 1; i <= 8744; ++i) cout << "128 ";
		cout << endl; cin >> v;
		if (v) {
			assert((r[v] - l[v] + 1) * 2 + 8744 <= 2.5e4);
			cout << "? " << (r[v] - l[v] + 1) * 2 << ' ';
			for (int i = l[v]; i <= r[v]; ++i)
				cout << l[v] << " " << i - l[v] + 1 << " ";
			cout << endl; cin >> v2; lm = r[v] - l[v] + 1;
			cout << "! " << r[v] - (v2 - lm - 1) << endl;
		}
		else {
			assert(127 * 128 + 8744 <= 2.5e4);
			cout << "? " << 127 * 128 << " ";
			for (int i = 1; i <= 127 * 128; ++i) cout << "1 ";
			cout << endl; cin >> v2; bool hs = 0;
			for (int i = 1; i <= 127; ++i)
				if (ceil(127.0 * 128 / i) == v2) {
					cout << "! " << i << endl;
					hs = 1; break;
				}
			assert(hs);
		}
	}
}

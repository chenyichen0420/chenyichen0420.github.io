思路分析

思维题的题解也不应该像天书，反倒更应该把思维链讲清楚。

我们看到这个题，感觉就不能直接模拟过程算，于是考虑有没有办法转换计数对象，显然是有的。

我们考虑对于每一段计算他的贡献，然而每一段的长度都不一样，不太好整，我们希望每一段的长度都一样，这样或许就可以全部放到一起计算了。

我们发现还真有方法，我们考虑假设操作序列是 $[(p_i,direction_i)]$，那么必定有一个等概率的操作序列是 $[n-p_i+1,reverse(direction_i)]$，也就是对称的情况。此时我们将两个操作序列的贡献相加，不难看出总贡献等于对长度全为 $\frac{d+d+(2n-1)x}{2}$ 的区间做同样的操作序列的贡献。

于是，问题转化成求初始间隔全部是 $\frac{d+d+(2n-1)x}{2}$ 的答案。

然后我应该怎么求答案呢？我们注意到所有的 $2n$ 段中每一段都有 $\frac{1}{2n}$ 的概率被选到，于是对答案的贡献就是总长度乘上 $\frac{1}{2n}$。

看起来我们要计数的对象已经变成了总长度！这显然是一个非常好的方向：尽可能地缩减状态数。于是我们考虑如何把总长度递推出来。

注意到总长度分为两种不同的变化方式：不变，或者左或右被砍掉一部分。而第二种情况会需要最边上两段的期望长度。

于是我们不妨换一个角度，从每一个区间的期望长度去考虑呢？

我们发现贡献变成了区间期望长度，这是好算的。而期望长度变化还是按照两种情况分讨如下：

  1. 砍掉最边上的两个区间，期望长度不变；
  2. 合并中间的两个区间，注意到合并区间的时候，被选到的那个区间一定会被合并到大区间，于是我们只在正好选中这个区间的时候统计其贡献，于是有期望长度变成 $\frac{n}{n-1}d$。

于是操作一次，区间期望长度就变成 $\frac{1}{n}\times d+\frac{n-1}{n}\times\frac{n}{n-1}\times d=\frac{n+1}{n}d$。

于是答案就可以求出来了，我们令第 $i$ 次操作之后期望长度为 $d_i$，有答案 $\sum_{i=0}^{d-1}d_i$，其中有 $d_0=\frac{2d+(2n-1)x}{2},d_i=\frac{n-i+2}{n-i+1}d_{i-1}=\frac{n+1}{n-i+1}d_0$。

此题可解，代码如下：

CPP 
#include<bits/stdc++.h>
using namespace std;
#define int long long
int n, x; double ans, d;
signed main() {
	ios::sync_with_stdio(0);
	cin >> n >> d >> x; d = (d * 2 + (2 * n - 1) * x) / 2;
	for (int i = 0; i != n; ++i) ans += d * (n + 1) / (n - i + 1);
	printf("%.10lf\n", ans);
}

唉，有些个题解我是真看了半天没看懂，拿着“期望线性性”的挡箭牌，然后式子和自己的组合意义对不上，思维题不是这样的！

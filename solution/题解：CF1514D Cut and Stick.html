思路分析

直觉上，我们只关心出现次数最多的数字的出现次数。

如果这个数字的出现次数不超过一半，那么一组就行。

如果超过一半，我们应该怎么用尽可能少的组把这个元素干掉呢？

显然，如果我们只看效率的话，$abb$ 这样的序列肯定能尽可能快的消耗完出现次数最多的数字，因此，如果出现次数最多的数字超过了剩下数字的两倍的话，那么答案就是剩下数字的个数加出现次数最多的数的出现次数减去剩下数字的个数的两倍。

还有一些情况就是出现次数最多的数比较多，但是没有上一种情况那么多，如果我仍然把这个数字给耗空的话就可能剩下一些原本比较少的元素，这样看起来不是很优秀。

考虑一种相对“妥协”了一下的决策：$aabbb$，我们不难想到只要我的出现次数最多的数字的出现次数只有剩余的数字的个数 $+1$，那么就可以一段直接端掉。

考虑到出现次数最多的那个数字的出现次数被卡在了这个区间内，因此我不停的进行 $abb$ 操作的话，一定存在一个时刻，正好满足现次数最多的数字的出现次数只有剩余的数字的个数 $+1$。

具体操作多少次？现次数最多的数字的出现次数减去剩余的数字的个数再减一。

此题可解，代码如下：

CPP 
#include<bits/stdc++.h>
using namespace std;
#define int long long
int n,m,a[300005],c[300005],mc,cc[300005],bs,ans[300005];
#define blk(p) (p/bs)
struct que{
	int l,r,id;
	inline bool operator<(const que&v)const{
		return blk(l)!=blk(v.l)?l<v.l:r<v.r;
	}
}q[300005];
inline void ins(int v){
	cc[c[v]]--; cc[++c[v]]++;
	if(mc<c[v]) mc=c[v];
}
inline void del(int v){
	cc[c[v]]--; cc[--c[v]]++;
	if(!cc[mc]) mc--;
}
signed main() {
	ios::sync_with_stdio(0); cin>>n>>m;
	for(int i=1;i<=n;++i) cin>>a[i];
	for(int i=1;i<=m;++i)
		cin>>q[i].l>>q[i].r,q[i].id=i;
	bs=sqrt(n); sort(q+1,q+m+1); cc[0]=n;
	/*
	直觉上，我们只关心出现次数最多的数字的出现次数
	如果这个数字的出现次数不到一半，那么一组就行
	剩下的，我们就要考虑什么样的东西是最优的了

	显然，论消耗速度的话，最优就是一拖二
	如果一拖二都不够用的话，那就只能剩一些单着的了
	否则，我们只需要一拖二拖到正好可以一半了

	那就直接莫队求一个出现次数最多的元素的出现次数就行了
	*/
	for(int l=1,r=0,i=1;i<=m;++i){
		while(r<q[i].r) ins(a[++r]);
		while(l>q[i].l) ins(a[--l]);
		while(r>q[i].r) del(a[r--]);
		while(l<q[i].l) del(a[l++]);
		int cnt=mc,ln=q[i].r-q[i].l+1,lft=ln-cnt;
		if(cnt<=lft+1) ans[q[i].id]=1;
		else if(cnt>=lft*2) ans[q[i].id]=lft+(cnt-lft*2);
		else ans[q[i].id]=(cnt-lft-1)+1;
	}
	for(int i=1;i<=m;++i) cout<<ans[i]<<"\n";
}

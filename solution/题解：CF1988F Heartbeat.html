思路分析

每一步都不难，拼到一起就让我卡住了。

首先，我们注意到一个性质：第一个后缀最大值和最后一个前缀最大值都是 $n$。

于是我们可以将最终的序列从 $n$ 的位置分开，统计前缀部分的贡献和后缀部分的贡献，然后拼起来得到最终的答案。

考虑怎么去拼这个东西，显然上升点并不是一个前后独立的状态，而前后部分的长度也是对答案有影响的，剩下的东西都是独立的，于是我们考虑让 $f_{i,j}$ 表示一个长度为 $i$ 的区间中有 $j$ 个上升点的所有排列的 $a$ 的和的值，$b_{i,j}$ 表示的是一个长度为 $i$ 的区间中有 $j$ 个上升点的所有排列的 $b$ 的和的值。

特别注意，因为前后缀的段实际上少统计了 $n$ 那个位置的贡献，因此实际计算 $f,b$ 的时候，如果有 $x$ 个最大值，那么实际用的应该是 $a_{x+1}$ 或 $b_{x+1}$。同时，我们还钦定 $f_{0,0}=b_{0,0}=1$。

显然，如果我知道了上面的两个数值，那么最终的答案就是 $f(i)=\sum_{j=1}^i\sum_{k1=0}^{j-1}\sum_{k2=0}^{i-j}f_{j-1,k1}b_{i-j,k2}c_{k1+k2+[j\neq1]}\binom{i-1}{j-1}$。

我们先考虑 $b,f$ 怎么计算，等会儿回过头来尝试优化这个 $O(n^4)$ 的答案计算。显然 $b,f$ 是类似的，于是我们只讲一个 $f$ 的计算方式，$b$ 留给读者自己推一下。

考虑 $f_{i,j}$ 显然信息并没有维护够：$a$ 并不是一个好递推的东西，于是我们再设一个辅助数组 $dp_{i,j,k}$ 表示长度为 $i$ 的，有 $j$ 个前缀最大值的，$k$ 个上升点的序列的数量。

可是，你仔细一想：这样的状态设计中没有存最终有多少个数，也难以维护一个值到底是不是最大值，于是我们考虑重新钦定一下转移顺序。

常见的转移顺序有两种：记录尾项排名的顺序递推和指定数值从大到小（或者从小到大）的递推。显然在这道题中如果使用第一种那么状态会额外多一维，大小为 $n$，无法接受，于是我们考虑第二种。

现在思考到底是从小到大还是从大到小。显然这道题中需要维护的是前缀最大值，如果从小到大处理那么会存在难以转移的问题，而从大到小则不存在这个问题，因此从大到小进行枚举。

考虑转移：只有将当前最小值放到第一个才能使得前缀最大值数量增加 $1$，同时原先的第一个位置成为了上升点，因此 $dp_{i,j,k}\to dp_{i+1,j+1,k+1}$。

如果将当前最小值放到一个上升点前，因为这个值比上升点的值小，所以上升点还是上升点，但是这个值也比之前上升点前的那个数值小，因此这个点本身没有成为上升点，因此 $dp_{i,j,k}\times k\to dp_{i+1,j,k}$。

其他的情况下，这个点应该能让他后面的点变成上升点，除非后面没有点，因此 $dp_{i,j,k}\to dp_{i+1,j,k}$，$dp_{i,j,k}\times(i-k-1)\to dp_{i+1,j,k}$。

显然状态数是 $O(n^3)$ 的，只有小常数种转移，因此这部分的时间复杂度是 $O(n^3)$ 的，对于这道题的时间限制来说问题不大。

$b$ 是类似的，在此跳过，我们考虑怎么去优化那个 $O(n^4)$ 的转移：

分析一下每一个因子的依赖项：四个不同的因子中分别使用到了 $(i,j),(j,k1),(i,j,k2),(j,k1,k2)$ 四组不同的变量。

显然最后我们是一定要枚举 $i$ 这个变量的，于是把 $i$ 视作常量，那么这几组变量就是 $([i],j),(j,k1),([i],j,k2),(j,k1,k2)$。注意到 $k1$ 和 $i$ 独立，于是我们考虑能不能通过枚举 $j,k1,k2$，将其中的 $k1$ 消掉，然后再枚举 $i,j,k2$ 计算答案。

我们发现确实可以，我们只需要预先计算出 $f^\prime_{j,k2}=\sum_{k1}\sum_{k2}f_{j,k1}c_{k1+k2+[j\neq1]}$，然后拿着 $f^\prime$ 和 $b$ 再进行转移就可以了，时间复杂度是 $O(n^3)$。

此题可解，代码如下：

CPP 
#include<bits/stdc++.h>
using namespace std;
#define int long long
constexpr int mod=998244353;
int n,a[1005],b[1005],c[1005],dp[2][1005][1005],cf[1005][1005],cb[1005][1005],ans,cvl[1005][1005],nf[1005][1005];
inline void mad(int& l,const int r){(l+=r)>=mod&&(l-=mod);}
signed main() {
	ios::sync_with_stdio(0); cin>>n;
	for(int i=1;i<=n;++i) cin>>a[i];
	for(int i=1;i<=n;++i) cin>>b[i];
	for(int i=0;i!=n;++i) cin>>c[i];
	for(int i=0;i<=n;++i)
		for(int j=cvl[i][0]=1;j<=i;++j)
			cvl[i][j]=(cvl[i-1][j]+cvl[i-1][j-1])%mod;
	dp[1][1][0]=1;
	for(int i=1;i!=n;++i){
		for(int j=1;j<=i;++j)
			for(int k=0;k!=i;++k)
				if(dp[i&1][j][k])
					//插入到最前面
					mad(dp[i+1&1][j+1][k+1],dp[i&1][j][k]),
					//插入到上升点前面或者最后，导致上升个数不变
					dp[i+1&1][j][k]=(dp[i+1&1][j][k]+dp[i&1][j][k]*(k+1))%mod,
					//其他情况
					dp[i+1&1][j][k+1]=(dp[i+1&1][j][k+1]+dp[i&1][j][k]*(i-k-1))%mod;
		for(int j=1;j<=i;++j)
			for(int k=0;k!=i;++k)
				cf[i][k]=(cf[i][k]+dp[i&1][j][k]*a[j+1])%mod;
		memset(dp[i&1],0,sizeof dp[0]);
	}
	memset(dp,0,sizeof dp);
	dp[1][1][0]=1;
	for(int i=1;i!=n;++i){
		for(int j=1;j<=i;++j)
			for(int k=0;k!=i;++k)
				if(dp[i&1][j][k])
					//插入到最后面
					mad(dp[i+1&1][j+1][k],dp[i&1][j][k]),
					//插入到上升点前面，导致上升个数不变
					dp[i+1&1][j][k]=(dp[i+1&1][j][k]+dp[i&1][j][k]*k)%mod,
					//其他情况
					dp[i+1&1][j][k+1]=(dp[i+1&1][j][k+1]+dp[i&1][j][k]*(i-k))%mod;
		for(int j=1;j<=i;++j)
			for(int k=0;k!=i;++k)
				cb[i][k]=(cb[i][k]+dp[i&1][j][k]*b[j+1])%mod;
		memset(dp[i&1],0,sizeof dp[0]);
	}
	cf[0][0]=a[1]; cb[0][0]=b[1];
	for(int j=1;j<=n;++j)
		for(int k1=0;k1<=j-1;++k1)
			for(int k2=0;k2<=n-j;++k2)
				nf[j-1][k2]=(nf[j-1][k2]+cf[j-1][k1]*c[k1+k2+(j!=1)])%mod;
	for(int i=1;i<=n;++i){
		for(int j=1;j<=n;++j)
			for(int k2=0;k2<=i-j;++k2)
				ans=(ans+nf[j-1][k2]*cb[i-j][k2]%mod*cvl[i-1][j-1])%mod;
		cout<<ans<<' '; ans=0;
	}
	cout<<endl;
}

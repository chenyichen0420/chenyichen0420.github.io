思路分析

好像在什么地方做过很类似的，总之是很经典的一个板题。

我们考虑一下怎么求解更改单个点之后的全局最大值。

显然，如果我强制指定你必须走过某一个点 (x,y)，那么答案就是从 (1,1) 走到 (x,y) 的最大值，加上从 (x,y) 走到 (n,m) 的最大值，最后减去 (x,y) 自己，因为被算了两次。

也就是说如果我枚举改了哪个点，那么经过这个点的最大值我就知道了。但显然我也可以选择不经过这个点，对吧？

我们再考虑一下能不能找到一个好处理的限制条件，使得我可以快速地求出不经过这个点的最优解？下面提供两种思路：

  1. 同一斜线上的点必定经过且只经过 1 个。

这里的斜线定义为 x+y 的值相等的点，上面所说的结论正确性是显然的。

于是我们枚举这个斜线，求出斜线上的最大值和次大值，然后顺次枚举换掉某一个点之后答案的最大值。

考场代码如下：

#include<bits/stdc++.h>
using namespace std;
#define int long long
int t, n, m, ans; vector<pair<int, int>>p, np;
vector<vector<int>>a, dp[2];
int v1, v2; pair<int, int>p1;
inline void ins(int v, pair<int, int> p) {
    if (v >= v1) v2 = v1, v1 = v, p1 = p;
    else if (v > v2) v2 = v;
}
signed main() {
    ios::sync_with_stdio(0);
    for (cin >> t; t; t--) {
        cin >> n >> m;
        a = dp[0] = dp[1] = vector<vector<int>>
            (n + 2, vector<int>(m + 2, 0xcfcfcfcfcfcfcfcf));
        for (int i = 1; i <= n; ++i)
            for (int j = 1; j <= m; ++j)
                cin >> a[i][j];
        dp[0][0][1] = dp[0][1][0] = 0;
        for (int i = 1; i <= n; ++i)
            for (int j = 1; j <= m; ++j)
                dp[0][i][j] = max(dp[0][i - 1][j], dp[0][i][j - 1]) + a[i][j];
        dp[1][n + 1][m] = dp[1][n][m + 1] = 0;
        for (int i = n; i >= 1; i--)
            for (int j = m; j >= 1; j--)
                dp[1][i][j] = max(dp[1][i + 1][j], dp[1][i][j + 1]) + a[i][j];
        p.emplace_back(1, 1); ans = 1e18;
        while (p.size()) {
            v1 = v2 = -1e18;
            for (const auto& pos : p)
                ins(dp[0][pos.first][pos.second] + dp[1][pos.first][pos.second] - a[pos.first][pos.second], pos);
            for (const auto& pos : p)
                if (pos == p1) ans = min(ans, max(v2, dp[0][pos.first][pos.second] + dp[1][pos.first][pos.second] - 3 * a[pos.first][pos.second]));
                else ans = min(ans, max(v1, dp[0][pos.first][pos.second] + dp[1][pos.first][pos.second] - 3 * a[pos.first][pos.second]));
            for (const auto& pos : p)
                if (pos.first != n) np.emplace_back(pos.first + 1, pos.second);
            if (p.back().second != m) np.emplace_back(p.back().first, p.back().second + 1);
            swap(p, np); np.clear();
        }
        cout << ans << endl;
    }
    return 0;
}
/*
给定一个 n*m 的矩阵 a，你可以选择一个元素并将其变为相反数。
求操作之后从 (1,1) 到 (n,m) 的路径的元素之和的最大值最小为多少。
这里的路径指的是走恰好 (n+m-2) 步的路径，也就是只能向右或者向下。
*/

事实上上述实现并不是特别优：显然只有更改最大值的那个位置严格优于其他的，我们只枚举那一个点就行，不过有一些细节，考场上没那么写。

  1. 经过右上角区域或左下角区域的覆盖所有不经过该点的路径。

思路分析

四川省集 Day4T2 搬的这道题，补完总结一下。

首先看到数据范围，十有八九能够猜到应该就是 $\log_2 n$ 了，毕竟 $\binom{C}{\frac{C}{2}}

当时磕了 $4h$ 这道题没有磕出来，到底卡在哪里了呢？

是这样的，我们不难有一个初步的思路就是广搜一下，然后让距离远的指向距离近的。然后我们就愉快的发现我们只需要处理好同距离的边就行了。

至于怎么指定一条边的方向，额，用你猫耳想一下都知道直接指明走到编号更大的还是编号更小的就行了。

可是，这样做真的可行吗？我们发现如果从每一个点出发，都有多个可行的出边的话，每一次真的之随便选一个边的话，是势必不可行的。

可惜可惜，考场上没有想到啊。。。

总之呢，我们认为，实际上点的编号也应该是一个有效的东西。

我们考虑这样一个方向：

原图实际上在经过广搜之后会形成一个最短路有向无环图，我们不妨从中生成出来一颗树，并尽可能的将任意一个点出发的位置驱赶到树上。

可是实践上这似乎几乎不可行：你并没有从根源上解决掉同层之间的转移问题。

我们不妨这样想：我们强制要求在最后选择节点的时候必须选择最小的可行节点。然后我们是不是就有一些操作空间了呢？毕竟，这样下来，最短路 DAG 立刻进化成为了最短路树。

我们思考对于同一层的点我们强制让最小的点最小的边最先纳入考虑就行了。

于是呢，我们现在考虑怎么样满足 $\log$ 的限制。一个非常合理的想法就是按秩合并，也就是子结点少的合并到子结点多的。合并完之后认为合并进去的那一个点的大小变成 $0$ 了。这样就可以防止一个点被重复统计，同时也不难发现这样子别的点不会进入到这个点来。

不过你可能会问：如果两个清 $0$ 的点之间需要给边定向，那你不就爆炸了吗？

可是我们注意到我们是按照最小的点最小的边进行排序的，一个点被清 $0$ 必然意味着其有一个更小的边，并且被定向出去了，所以 $0$ 内部的连边最终一定不会被选择。

于是此题可解，代码如下：

CPP 
#include<bits/stdc++.h>
using namespace std;
int d[10005],sz[10005]; queue<int>q; bool vis[50005];
vector<pair<int,int>>son[10005];
struct node{int l,r,md,id;}; vector<node>ev;
vector<int>label(int n,vector<pair<int,int>>e,int t){
	for(int i=0;i!=e.size();++i)
		son[e[i].first].emplace_back(e[i].second,i),
		son[e[i].second].emplace_back(e[i].first,i);
	q.emplace(t); d[t]=1;
	while(q.size()){
		int p=q.front();q.pop();
		for(const auto& sp:son[p]) if(!d[sp.first])
			d[sp.first]=d[p]+1,q.emplace(sp.first);
	}
	for(int i=1;i<=n;++i) sz[i]=1;
	vector<int>ret(e.size(),0);
	for(int i=0;i!=e.size();++i)
		ev.push_back(node{
			min(e[i].first,e[i].second),
			max(e[i].first,e[i].second),
			max(d[e[i].first],d[e[i].second]),
			i
		});
	sort(ev.begin(),ev.end(),[](const node&l,const node&r){
		if(l.md!=r.md) return l.md>r.md;
		if(l.r!=r.r) return l.r<r.r;
		return l.l<r.l;
	});
	for(auto&v:ev){
		if(d[v.l]!=d[v.r]){
			if(d[v.l]>d[v.r]) swap(v.l,v.r);
		}
		else{
			if(sz[v.l]<sz[v.r]) swap(v.l,v.r);
		}
		ret[v.id]=v.l>v.r;
		sz[v.l]+=sz[v.r]; sz[v.r]=0;
	}
	return ret;
}
int travel(int n, int node, vector<pair<int, int>> directions){
	sort(directions.begin(), directions.end());
    for(const auto&v:directions){
        if(v.first<node&&v.second==0) return v.first;
        if(v.first>node&&v.second==1) return v.first;
    }
	return -1;
}

思路分析

说实话，就这种题，不证明正确性等于白干。思路很简单啊，从右下向左上枚举，发现不同就将左上的全部更改。

但是，正确性存疑：如果出现这样三个坐标 P(x1​,y1​),A(x2​,y1​),B(x1​,y2​)，满足坐标本身合法且 x1​<x2​,y1​<y2​。这时候 A,B 位置为 B，C 位置为 W，那么先枚举 A 和先枚举 B 就会产生差别。

然而事实真的都是这样的吗？

并不是。这时候如果你对点 O(x2​,y2​) 进行分讨的话，你就会发现这并无影响。

代码如下：

#include <bits/stdc++.h>
using namespace std;
int n, m, ans[105][105], ant[105][105], cnt; char c;
inline void chg(const int& px, const int& py, const int& cv) {
	for (int i = px; i; i--)
		for (int j = py; j; j--)
			ant[i][j] += cv;
}
signed main() {
	ios::sync_with_stdio(0);
	cin >> n >> m;
	for (int i = 1; i <= n; ++i)
		for (int j = 1; j <= m; ++j)
			ans[i][j] = (cin >> c, c);
	for (int i = n; i; i--)
		for (int j = m; j; j--)
			if (ans[i][j] ^ ant[i][j])
				chg(i, j, ans[i][j] - ant[i][j]), cnt++;
	cout << cnt << endl;
	return 0;
}

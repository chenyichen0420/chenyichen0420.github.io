思路分析

不少题解是直接给出划线的做法，然后证明正确，非常神奇。

我们考虑什么样的操作是合法的：

$01x01$ 中的 $x$ 是可以变化的 $10x10$ 中的 $x$ 也是可以变化的。

实际上好像也就没有别的情况了。

但是这个东西好像不是很好刻画，我们随便挑一种来看一下：

$01001\to01101$ 可以理解为长度为 $1$ 的段和长度为 $2$ 的段换了个位置。

有没有特殊情况？在边界上可能出现 $2$ 拆分成两个 $1$ 或者两个 $1$ 合并成一个 $2$ 的情况。

于是给人的感觉就像是区间的边界在进行调整，也就有了那个划线的做法。

具体来说，我们考虑维护 $s,t$ 的 $01$ 发生变化的那些分界点，然后要求分界点不可重（串外面的我们不管）。我们尝试让 $s$ 和 $t$ 的分界点重合。

于是显然我们枚举 $01$ 相同的分界点进行暴力就行了，时间复杂度是 $O(n^2)$ 的。

代码如下：

CPP 
#include<bits/stdc++.h>
using namespace std;
#define int long long
int n, sc, tc, ans = 1e9, cs[5005], ct[5005], ant;
char s[5005], t[5005];
inline int at(int p, int* c, int s) {
    if (p <= 0) return 0;
    if (p > s) return n;
    return c[p];
}
inline int sat(int p) { return at(p, cs, sc); }
inline int tat(int p) { return at(p, ct, tc); }
signed main() {
    ios::sync_with_stdio(0); cin >> n >> (s + 1) >> (t + 1);
    for (int i = 1; i < n; i++) if (s[i] != s[i + 1]) cs[++sc] = i;
    for (int i = 1; i < n; i++) if (t[i] != t[i + 1]) ct[++tc] = i;
    for (int i = -sc - 1; i <= tc + 1; i++)
        if (i % 2 && s[1] != t[1] || i % 2 == 0 && s[1] == t[1]) {
            for (int j = 1 - max(0ll, i); j <= max(sc, tc - i); j++)
                ant += abs(sat(j) - tat(i + j));
            ans = min(ans, ant); ant = 0;
        }
    cout << ans << endl;
}

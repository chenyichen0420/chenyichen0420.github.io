思路分析

相信大部分人的第一思路都是深搜什么的，然后 (2×105)2(2\times10^5)^2(2×105)2 当场爆炸。

不过我们发现如果我们能够八连通的画出这么一道只经过障碍的线，使其从左边界或下边界连到右边界或者上边界，那他一定就被分开了。反之则一定没有分开。

正着想其实没啥大的问题，你甚至可以画画图或者四种情况分讨一下就能出来。反之则因为它可以四联通的到处钻洞，一定能够找到一条路。

有没有这样一条路径呢？深搜吗？其实我们发现这条线没有多高的要求，他甚至不需要是一条简单路径，是、只要能够连上两边就行了。所以，并查集是一个不错的选择。

代码如下：

CPP 
#include <bits/stdc++.h>
using namespace std;
#define int long long
int n, m, k, l[200005], r[200005], f[200005]; 
map<pair<int, int>, int>id;
constexpr int
dx[] = { 0,0,1,1,1,-1,-1,-1 },
dy[] = { 1,-1,0,1,-1,0,1,-1 };
inline int find(int p) {
	return f[p] != p ? f[p] = find(f[p]) : p;
}
inline void merge(int l, int r) { f[find(l)] = find(r); }
signed main(){
	ios::sync_with_stdio(0);
	cin >> n >> m >> k;
	for (int i = 1; i <= n; ++i)
		id[make_pair(i, 0)] = 1,
		id[make_pair(i, m + 1)] = 2;
	for (int i = 1; i <= m; ++i)
		id[make_pair(0, i)] = 2,
		id[make_pair(n + 1, i)] = 1;
	for (int i = 1; i <= k; ++i)
		cin >> l[i] >> r[i],
		id[make_pair(l[i], r[i])] = i + 2;
	for (int i = 1; i <= k + 2; ++i) f[i] = i;
	for (int i = 1; i <= k; ++i)
		for (int j = 0; j != 8; ++j)
			if (id.count(make_pair(l[i] + dx[j], r[i] + dy[j])))
				merge(i + 2, id[make_pair(l[i] + dx[j], r[i] + dy[j])]);
	cout << (find(1) == find(2) ? "No\n" : "Yes\n");
}

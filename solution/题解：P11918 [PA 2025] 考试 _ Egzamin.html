思路分析

四川省集最后一天讲课的时候 251Sec 将了这道题，感觉非常的牛啊，写一篇题解。

显然最优解就是将 $p_i$ 从大到小排序选一个前缀作答。于是我们令 $f_{i,j}$ 表示回答完前 $i$ 道题之后分数是 $j$ 的概率。不难发现这个东西直接做的时间复杂度是 $O(n^2)$ 的。

但是吧，感觉上如果某一个 $f_{i,j}\le eps$，那么保留他似乎并不会对最终的答案造成太大的影响。

于是我们设置一个小一点的阈值，假设最终要求的精度为 $eps_0$，那么我们设置阈值为 $\frac{eps_0}{n}$ 就行。我们发现通过了。

考虑证明区间长度的期望界限。上课的时候通过使用并证明了一个结论：$n$ 个独立的以 $0.5$ 的概率取道 $-1,1$ 的随机变量和 $>\sqrt{2n\log(\frac{1}{eps})}$ 的机率 $

但是客观的说，我认为这可能并不是一个好的思考方向，为了使用这个结论的话，会需要大量的说明，尤其是那个 $0.5$ 的概率和这道题说真的并没有感觉到，有什么好处。

因此我们考虑还一个角度：怎么样做能够尽可能拉长有效值的分布区间？

显然，我们不会左右横跳，也就是说我们可以只考虑所有的概率都 $>0.5$ 的情况。

另外，我们考虑如果所有的概率都靠近 $0.5$，就会使得数值摊的太平，很快就会全部失效，因此这样的值可以忽略，他们只带来常数差距。

如果所有的概率都过于靠近 $1$，就会导致数值并没有全局摊开，甚至一分裂出去就失效了，所以也不行。

于是我们有一个直觉：似乎，我让他开始的时候快速分裂，等分裂到最大值的时候在一路 $1$ 下去就非常的优秀了。

然后我们思考什么样的 $p$ 能够分裂到最大值，能够摊的最平。我们考虑别的 $p$ 和 $0.5$ 进行比较：

如果是 $0.5$，那么左右一路对称走下去，直到到达边界，使得 $f_{min}\ge eps>\frac{f_{min}}{2}$，但是相反别的 $p$ 会导致值向一侧倾倒，实际上，如果略微偏移一点点可以让数值更加集中，有利于保持数值。

最终，我们通过三分，发现 $p=0.507908$ 时宽度达到了最大值 $2691$，不难发现在此题的数据范围下还是可以通过的。

参考代码如下：

CPP 
#include<bits/stdc++.h>
using namespace std;
#define int long long
#define double long double
constexpr double eps=1e-12;
int n,t,l,r,o; double dp[2][100005],p[50005],ans,ant;
signed main(){
	ios::sync_with_stdio(0); cin>>n>>t;
	for(int i=1;i<=n;++i) cin>>p[i];
	sort(p+1,p+n+1,greater<double>());
	l=r=0; dp[0][0]=1;
	for(int i=1;i<=n;++i){
		for(int j=l;j<=r;++j)
			dp[o^1][j]+=dp[o][j]*(1-p[i]),
			dp[o^1][j+2]+=dp[o][j]*p[i],
			dp[o][j]=0;
		r+=2; o^=1; ant=0;
		while(dp[o][l]<=eps) dp[o][l++]=0;
		while(dp[o][r]<=eps) dp[o][r--]=0;
		for(int j=t+i;j<=r;++j) ant+=dp[o][j];
		ans=max(ans,ant);
        assert(r-l<=80);
		// cerr<<i<<" "<<l<<":\n";
		// for(int j=l;j<=r;++j) cerr<<dp[o][j]<<"\n";
	}
	printf("%.10Lf\n",ans);
}

思路分析

一个错误的贪心思路是投自己家所在的方向，样例一就给你卡了。

我们思考这个东西为什么会错：自己这边的势力确实可能短暂强于对方，但是一旦自己这边的一个巨佬提前到家了，那自己的优势就没有了，又会被别人拽一边去，得不偿失。

感觉正着考虑没有前途，于是正难则反，我们思考最终的状态是什么。

首先最后到的位置一定是 $1,n$ 中的某一个，显然如果到达这两个点则一定到达过中间的所有点。

这个问题是简单的：谁少谁等着。

我们不妨假设最后到的是 $1$，那么在此之前最后到的一定是 $2,n$，证明同上，发现问题相似，但不完全一样：此时要到达 $1$ 的那些人也是要参与投票的。

于是我们考虑作为 $1$ 我会干什么：肯定死命往 $n$ 那边送，对吧？也就是 $2,n$ 争先后的时候我们的 $1$ 会想要让车尽量先到 $n$ 那边去，再一口气滚回来。

好的，我们不妨假设 $n$ 此时得的票是仍然不够的，也就是倒数第二个到的是 $n$，那么问题变成了 $2,n-1$ 中选一个最后到。

显然，目前的目标变成了尽快的先到达 $n$，于是 $1,n$ 的决策其实都是一致的：给 $2$ 投，让车尽快到 $2$ 了后一路滚回 $n$。

至此，策略已经分析完了，边界情况就是初始位置不在决策区间内，此时按照方向移动过去即可。

用双指针模拟上述过程可以做到线性，代码如下：

CPP 
#include<bits/stdc++.h>
using namespace std;
#define int long long
int n, m, x[100005], p[100005], mid, l, r;
vector<int>op; int cp, ans, cnt, dir;
signed main() {
	ios::sync_with_stdio(0);
	cin >> n >> m; l = 1, r = n; cp = m; dir = 1;
	for (int i = 1; i <= n; ++i) cin >> x[i] >> p[i];
	mid = upper_bound(x + 1, x + n + 1, m) - x;
	while (l < mid && r >= mid)
		if (p[l] + cnt * dir < p[r])
			op.emplace_back(l),
			cnt += p[l++], dir = -1;
		else
			op.emplace_back(r),
			cnt += p[r--], dir = 1;
	while (l < mid) op.emplace_back(l++);
	while (r >= mid) op.emplace_back(r--);
	reverse(op.begin(), op.end());
	for (int i : op) ans += abs(cp - x[i]), cp = x[i];
	cout << ans << endl;
}

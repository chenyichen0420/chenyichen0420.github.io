思路分析

大家的思路似乎有点复杂了。直接并查集加上暴力跳树时间复杂度是显然正确的，实现极为简单。可以降绿了，真的没有什么难点。

首先我们知道除法向下取整的话是可以任意交换计算顺序的。也就是实现的时候可以选择两边向中间靠拢，边靠边计算，实现更简洁。

其次我们知道如果除的数不是 111，那么至少会砍半（废话），因此如果结果不是 000，那么中间最多经过了不超过 log⁡2V\log_2Vlog2​V 条非一边。

然后我们也不难发现经过数值为一的边不会影响结果，因此我们可以将数值为一的边连接的两部分直接视为一个大点。

最后题目保证修改边权操作只减不增，也就是说数值为一的边不会发生改变。也就是说只会合并不会分裂，可以直接使用并查集维护。

然后就做完了，注意并查集只能深度深的点合并到深度浅的点上。时间复杂度是 O(nlog⁡V)O(n\log V)O(nlogV) 的。

代码压行较为严重，仅 323232 行。仅供参考。

CPP 
#include<bits/stdc++.h>
using namespace std;
#define int long long
struct node {
	int p, v; node(int p = 0, int v = 0) :p(p), v(v) {};
}e[200005]; vector<node>son[200005];
int n, m, f[200005], jmp[200005], dep[200005], a[200005];
inline int find(int p) { return jmp[p] != p ? jmp[p] = find(jmp[p]) : p; }
inline void dfs(int p, int fa) {
	f[p] = fa; dep[p] = dep[fa] + 1;
	for (const node& sp : son[p]) if (sp.p != fa)
		(a[sp.p] = sp.v) == 1 && (jmp[sp.p] = p), dfs(sp.p, p);
}
inline void cal(int l, int r, int& v) {
	while (l != r) {
		if (dep[l] < dep[r]) swap(l, r);
		if (!(v /= a[l])) return; l = find(f[l]);
	}
}
signed main() {
	ios::sync_with_stdio(0); cin >> n >> m;
	for (int i = 1; i <= n; ++i) jmp[i] = i;
	for (int i = 1, l, r, v; i != n; ++i)
		cin >> l >> r >> v, e[i] = { l,r },
		son[l].emplace_back(r, v),
		son[r].emplace_back(l, v);
	dfs(1, 0);
	for (int i = 1, o, l, r, v; i <= m; ++i)
		if (cin >> o >> l >> r, o & 1) cin >> v, cal(l, r, v), cout << v << endl;
		else if (f[e[l].p] == e[l].v) ((a[e[l].p] = r) == 1) && (jmp[e[l].p] = e[l].v);
		else ((a[e[l].v] = r) == 1) && (jmp[e[l].v] = e[l].p);
}

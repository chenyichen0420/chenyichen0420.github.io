思路分析

非常简单的一道题。

首先我们假设我们可以随便切割，那么答案最大是多少？答案很简单，和定差小积大，答案为 ⌊2c​⌋×⌈2c​⌉。

考虑能不能构造出来方案，提供两种思路：

  1. 逐步调整

我们考虑凑个数。比方说我现在要凑 ⌊2c​⌋ 个在某一半，那我就从上往下扫。

如果加上这一行仍然不到，那就直接加上这一行（对应 D 操作），如果能到，那就依次加进来直到个数足够。显然时间复杂度是 O(n+m) 的。

考场代码如下，实现的就是一坨：

#include<bits/stdc++.h>
using namespace std;
#define int long long
int t, n, m, cnt, cnr[300005];
vector<vector<char>>c;
signed main() {
    ios::sync_with_stdio(0);
    for (cin >> t; t; t--) {
        cin >> n >> m; cnt = 0;
        c = vector<vector<char>>(n + 1, vector<char>(m + 1, ' '));
        for (int i = 0; i != n; ++i) cnr[i] = 0;
        for (int i = 0; i != n; ++i)
            for (int j = 0; j != m; ++j)
                if (cin >> c[i][j], c[i][j] ^ 48)
                    cnr[i]++, cnt++;
        for (int i = 1; i != n; ++i) cnr[i] += cnr[i - 1];
        cout << cnt / 2 * (cnt + 1 >> 1) << endl;
        for (int i = n - 1; i >= 0; i--)
            if (i <= 0 || cnr[i - 1] <= (cnt + 1) / 2) {
                cnt = (cnt + 1) / 2 - (i > 0 ? cnr[i - 1] : 0);
                int pos = m; //cerr << i << endl;
                for (int j = m - 1; j >= 0; j--)
                    if (c[i][j] == '1' && (--cnt == 0))
                        pos = j;
                //cerr << pos << endl;
                for (int j = 0; j != i; ++j) cout << "D";
                for (int j = 0; j != pos; ++j) cout << "R";
                cout << "D";
                for (int j = pos; j < m; ++j) cout << "R";
                for (int j = i + 1; j < n; ++j) cout << "D";
                break;
            }
        cout << endl;
    }
    return 0;
}

RDRDRDRDDR
DRRRDRDDD

  1. 逐步扩张

我们考虑将所有的 1 按照到 (1,m) 的距离从小到大排序。如果距离一样就按照 x 轴从大到小。这里的距离定义为曼哈顿距离。最后取出前 2c​ 个进行切割即可。

为什么是正确的呢？首先，如果选择了一个点，那就意味着他和右上角之间形成了一个矩形，其中没有不被选择的 1，于是形成一个源自右上角的凸包，显然存在对应的操作序列。

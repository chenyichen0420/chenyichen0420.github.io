首先给出结论，令 ，显然有 经过 次 操作后变为 。那么一定有答案为 。

为什么呢？

首先，我们知道，当你将一个非零值乘三之后，你的操作次数必然至少会加一，因为你还得通过除以三的操作将他变小。

因此，我们肯定会先将一个数变为零，然后其他的数用这个数一直除以三。

这个数显然取最小的那个数。因为你会多做 次操作， 的时候这个值一定是 中的最小值。

但是我们还有一个问题：快速求解 。两种方法：可以全部计算，然后求前缀和。这种方法当然极其快，而且常数小。

可是我脑抽了，竟一时间没有想到，转而采用了另一种方式。类似于除法分块。我们把 值相同的 放在一起。显然可也证明这是连续且单增的。分界点就是 。

这种方法对应的代码如下，当然并不建议大家使用：

#include<bits/stdc++.h>
using namespace std;
int t,l,r,tri[15],pl,pr,ans;
int main(){
    ios::sync_with_stdio(0);
    cin>>t; tri[1]=1;
    for(int i=2;i<=14;++i) tri[i]=tri[i-1]*3;
    while(t--){
        cin>>l>>r;
        pl=upper_bound(tri+1,tri+15,l)-tri-1;
        pr=upper_bound(tri+1,tri+15,r)-tri-1;
        cerr<<pl<<" "<<pr<<endl;
        if(pl==pr){
            ans=(r-l+1)*pl;
            cout<<ans+pl<<endl;
            continue;
        }
        ans=(tri[pl+1]-l)*pl;
        ans+=(r-tri[pr]+1)*pr;
        for(int i=pl+1;i<pr;++i)
            ans+=(tri[i+1]-tri[i])*i;
        cout<<ans+pl<<endl;
    }
    return 0;
}

思路分析

首先设 $dp_i$ 为表示到 $i$ 的最小花费，考虑转移，显然是 $dp_i=\min dp_j+val_{j+1,i}$，其中 $val_{j+1,i}$ 是用单个字符串表示 $s_{j+1,i}$ 这个子区间的最小长度。

显然，如果我选择 $s_{j+1,i}$ 的最短循环节作为我的内容的话是一定不劣的，因为字符串长度是除以次数的，而次数占据的字符数是 $\log$ 的。

于是问题变成了求解所有子区间的最短循环节。我们掏出 border 的美妙性质：一个字符串的最短循环节是串长减去整串的 border 的长，如果确实存在的话。检验方法就是看整个串的长度是不是你算出来的循环节的长度的倍数。

一个简短的较为严谨的证明：border 的长度 $b$ 对应周期 $p=n-b$。若 $p\mid n$，整串由 $p$ 长前缀重复而成，$p$ 最小即 $b$ 最长，故为最短循环节。

于是直接模拟上述过程就行了，时间复杂度 $O(n^2)$，参考代码如下：

CPP 
#include<bits/stdc++.h>
using namespace std;
// #define int long long
char s[8005]; int n,l[8005],dp[8005],c[8005][8005],ln[8005];
signed main(){
	ios::sync_with_stdio(0);
	cin>>s; n=strlen(s);
	memmove(s+1,s,n*sizeof(char));
	for(int i=1;i<=n;++i){
		memset(l,0,sizeof l);
		for(int j=2,k=0;j<=n-i+1;++j){
			while(k&&s[j]!=s[k+1])k=l[k];
			l[j]=(k+=(s[j]==s[k+1]));
		}
		for(int j=i;j<=n;++j){
			int ln=j-i+1-l[j-i+1];
			if((j-i+1)%ln) c[i][j]=j-i+1;
			else c[i][j]=ln;
		}
		memmove(s,s+1,n*sizeof(char));
	}
	for(int i=1;i<=8000;++i) ln[i]=ln[i/10]+1;
	memset(dp,0x0f,sizeof dp); dp[0]=0;
	for(int i=1;i<=n;++i)
		for(int j=0;j!=i;++j)
			dp[i]=min(dp[i],dp[j]+c[j+1][i]+ln[(i-j)/c[j+1][i]]);
	cout<<dp[n]<<endl;
}

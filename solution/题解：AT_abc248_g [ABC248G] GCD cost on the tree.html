思路分析

感觉这种对 $gcd$ 什么的容斥方法已经非常的普及了。

就是这样的：我们考虑将对 $=k$ 的贡献转换成 $\mid k$ 的贡献，然后把其倍数给砍掉就是 $=k$ 的贡献了。

在这道题中，我们考虑统计 $f(d)=\sum_{d|\gcd(i,j)}dis(i,j)$ 的值，然后就有答案是 $\sum_{i\ge1}d(f(d)-\sum_{j\ge2}f(jd))$。考虑 $f(d)$ 怎么计算。

显然对 $f(d)$ 有贡献的路径 $(i,j)$，满足 $\forall k\in(i,j)$，$d|a_k$。

于是我们将所有的因数的出现位置记出来，然后这样的路径就是若干个连通块内部的路径。接下来考虑对于一个连通块，怎么统计出所有路径的长度。

显然，这些连通块仍然是树形结构，于是我们考虑我们要计算的就是一个子树中的所有路径的长度之和。

不妨遍历这个树上的每一个节点，统计子树中经过这个节点的路径的长度之和，这是简单的，我们记录出每一颗子树到这个节点的路径长度和以及个数，然后就可以简单计算出来了。

完了之后按照前面给的公式直接计算就行了，时间复杂度是 $n\sigma_0(n)+n\log n$，可以通过此题，参考代码如下：

CPP 
#include<bits/stdc++.h>
using namespace std;
#define int long long
constexpr int mod=998244353;
int n,a[100005],ans[100005],c[100005],v[100005],nc[100005],ret;
vector<int>son[100005],ap[100005]; bool cs[100005];
inline void mad(int& l,const int r){(l+=r)>=mod&&(l-=mod);}
inline void dfs(int p,int a){
	c[p]=v[p]=1; nc[p]=cs[p]=0;
	for(int sp:son[p]) if(cs[sp]){
		dfs(sp,a);
		nc[p]=(nc[p]+c[p]*v[sp]+c[sp]*v[p])%mod;
		mad(c[p],c[sp]); v[p]=(v[p]+v[sp]+c[sp])%mod;
	}
	mad(ans[a],nc[p]);
}
signed main(){
	ios::sync_with_stdio(0); cin>>n;
	for(int i=1;i<=n;++i){
		cin>>a[i];
		for(int j=1;j*j<=a[i];++j)
			if(a[i]%j==0) ap[j].emplace_back(i);
		for(int j=1;j*j<a[i];++j)
			if(a[i]%j==0) ap[a[i]/j].emplace_back(i);
	}
	for(int i=1,l,r;i!=n;++i)
		cin>>l>>r,
		son[l].emplace_back(r),
		son[r].emplace_back(l);
	for(int i=1e5;i>=1;i--) if(ap[i].size()){
		for(int p:ap[i]) cs[p]=1;
		for(int p:ap[i]) if(cs[p]) dfs(p,i);
		for(int v=i*2;v<=1e5;v+=i) mad(ans[i],mod-ans[v]);
	}
	for(int i=1;i<=1e5;++i) ret=(ret+i*ans[i])%mod;
	cout<<ret<<endl;
}
//私は猫です!
//Written by cbc on nyarch linux.

注：上面时间复杂度中的 $\sigma_0$ 实际上表示的是 $1\sim n$ 中的所有数字的因数个数的最大值，在 $10^5$ 范围内只有大约 $130$，不大。

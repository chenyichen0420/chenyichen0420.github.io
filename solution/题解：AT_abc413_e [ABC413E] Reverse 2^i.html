思路分析

类似于归并排序。

一个显然的结论：我们可以将任意一个长度为 2k2^k2k 的区间的一个特定的数移动到最前面。每一步只要这个数在后半段就把他移到前半段去就完事了。

这启示我们对于每一步，我们都先判断最小的数在前半段还是后半段。在后半段就翻转一下，然后递归下去处理两个子区间。

不难发现由于字典序靠前的数的影响力严格大于后面的任意一堆数，所以这样子达到了最优。

代码如下：

CPP 
#include <bits/stdc++.h>
using namespace std;
#define int long long
int t, n, a[1ll << 18 | 2];
inline void tmin(int& l, const int r) { (l > r) && (l = r); }
inline int gmin(int l, int r) {
	int ret = a[r];
	for (int i = l; i != r; ++i) tmin(ret, a[i]);
	return ret;
}
inline void msort(int l, int r) {
	if (l == r) return; int mid = l + r >> 1;
	int ml = gmin(l, mid), mr = gmin(mid + 1, r);
	if (ml > mr) reverse(a + l, a + r + 1);
	msort(l, mid); msort(mid + 1, r);
}
signed main(){
	ios::sync_with_stdio(0);
	for (cin >> t; t; t--) {
		cin >> n;
		for (int i = 1; i <= 1ll << n; ++i) cin >> a[i];
		msort(1, 1ll << n);
		for (int i = 1; i <= 1ll << n; ++i) cout << a[i] << " ";
		cout << endl;
	}
}

难度应该就是中位黄吧。

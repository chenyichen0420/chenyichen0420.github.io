思路分析

当你没有思路的时候就要去看数据范围。

我们注意到题目中保证了 nnn 的和不会超过 2×1052\times10^52×105。猜测是线性的。

这时候我们换一个思路。既然我们不能够直接枚举子序列，那我们可不可以间接枚举呢？

我们注意到，只有这个子序列中的中位数为 111，也就是说 111 的个数大于等于 k+12\frac{k+1}{2}2k+1​ 时才对答案有贡献。

同时我们知道，111 和 000 的具体位置我们并不关心。我们只需要有那么多个 111 和剩下的那么多个 000 就够了。

思路顺势就出来了：排列组合。

首先枚举子序列中有多少个 111。假设子序列中有 ccc 个 111，原序列中有 ttt 个 111，可能的情况数为 CtcC_t^cCtc​。然后考虑 000。子序列中有 k−ck-ck−c 个，原序列中有 n−tn-tn−t 个，可能的情况数为 Cn−tk−cC_{n-t}^{k-c}Cn−tk−c​。因此，子序列中有 ccc 个一的情况数为 Ctc×Cn−tk−cC_t^c\times C_{n-t}^{k-c}Ctc​×Cn−tk−c​。

枚举一下答案就出来了。注意 c≥k+12c\ge\frac{k+1}{2}c≥2k+1​。

代码如下：

CPP 
#include<bits/stdc++.h>
using namespace std;
#define int long long
constexpr int mod=1e9+7,lim=2e5;
int t,n,m,a,jc[200005],ny[200005],cnt,ans;
inline int cvl(const int&n,const int&m){
	if(m>n) return 0;
	return jc[n]*ny[m]%mod*ny[n-m]%mod;
}
inline int qpow(int a,int b=mod-2,int p=mod){
	int tmp=1;
	while(b){
		if(b&1) tmp*=a,tmp%=p;
		a*=a; a%=p; b>>=1;
	}
	return tmp;
}
signed main(){
	ios::sync_with_stdio(0);
	cin>>t; jc[0]=1;
	for(int i=1;i<=lim;++i) jc[i]=jc[i-1]*i%mod;
	ny[lim]=qpow(jc[lim]);
	for(int i=lim;i>0;i--) ny[i-1]=ny[i]*i%mod;
	while(t--){
		cin>>n>>m; ans=cnt=0;
		for(int i=1;i<=n;++i)
			cin>>a,cnt+=a;
		for(int i=m/2+1;i<=m&&i<=cnt;++i)
			ans+=cvl(cnt,i)*cvl(n-cnt,m-i),
			ans%=mod;
		cout<<ans<<endl;
	}
	return 0;
}

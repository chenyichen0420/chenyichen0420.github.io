思路分析

典型的带有单调性的问题，考虑进行二分。

我们思考一下，发现如果一条边是路径上的 $k$ 大值，那我们可以考虑将比它大的边的边权重新标记为 $1$，剩下的标记为 $0$。

这样一来，就会存在一条路径，满足权值和为 $k-1$。显然也就对应着有 $k-1$ 个比这条边更大的，即这条边是第 $k$ 大的。

因此我们可以选择从小到大依次删边，跑 floyd 全源最短路，然后询问时进行二分。

逻辑上有一点点瑕疵，就是他的第 $k$ 大边有可能和 $k-1$ 大边边权一样，在前文的重新标号中不会认为路径权值和是 $k-1$，但是实现的时候是一条一条删的，也就是即使同样的边权也会呈现逐个递减的数值变化，覆盖所要求的值。

这样做时间复杂度是 $O(mn^3+q\log n)$ 的，不可接受。

我们注意到每一次都只会改变一条边的状态，所以我们在跑出来初始状态之后每次只需要针对这条边更新一下点对距离就行了。时间复杂度 $O(n^3+mn^2+q\log n)$，还是有点高。

我们注意到每一次更改都是将边权调整为 $0$，而仅由边权为 $0$ 的边所构成的连通块的内部更改不在会对全局造成任何影响，因此实际有影响的更改最多只会有 $n-1$ 次。时间复杂度 $O(n^3+q\log n)$，可以通过。

代码如下：

CPP 
#include<bits/stdc++.h>
using namespace std;
struct node { int l, r, v; }e[100005];
int t, n, m, k, d[405][405][405], v[405], vc;
inline void tmin(int& l, const int r) { (l > r) && (l = r); }
signed main() {
	ios::sync_with_stdio(0);
	for (cin >> t; t; t--) {
		cin >> n >> m >> k; vc = 0;
		memset(d, 0x0f, sizeof d);
		for (int i = 1, l, r, v; i <= m; ++i)
			cin >> l >> r >> v, e[i] = { l,r,v },
			d[0][l][r] = d[0][r][l] = 1;
		for (int i = 1; i <= n; ++i) d[0][i][i] = 0;
		for (int k = 1; k <= n; ++k)
			for (int i = 1; i <= n; ++i)
				for (int j = 1; j <= n; ++j)
					tmin(d[0][i][j], d[0][i][k] + d[0][k][j]);
		sort(e + 1, e + m + 1, [](const node& l, const node& r) {return l.v < r.v; });
		for (int i = 1; i <= m; ++i) {
			if (!d[vc][e[i].l][e[i].r]) continue; v[++vc] = e[i].v;
			for (int j = 1; j <= n; ++j) for (int k = 1; k <= n; ++k)
				tmin(d[vc][j][k], d[vc - 1][j][k]),
				tmin(d[vc][j][k], d[vc - 1][j][e[i].l] + d[vc - 1][e[i].r][k]),
				tmin(d[vc][j][k], d[vc - 1][j][e[i].r] + d[vc - 1][e[i].l][k]);
		}
		for (int i = 1, l, r, v; i <= k; ++i) {
			cin >> l >> r >> v; int lp = 0, rp = vc, mid;
			while (lp != rp) {
				mid = lp + rp >> 1;
				if (d[mid][l][r] >= v) lp = mid+1;
				else rp = mid;
			}
			cout << ::v[lp] << ' ';
		}
		cout << endl;
	}
}

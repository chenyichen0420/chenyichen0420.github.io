思路分析

拿到这个题，差不多就可以盲猜长度是 O(log⁡n)O(\log n)O(logn) 的了。

直觉上是成立的。我们这样思考：假设我只选 ⌊n3⌋+1∼n\lfloor\frac{n}{3}\rfloor+1\sim n⌊3n​⌋+1∼n 之间的数，显然他们的 gcd⁡\gcdgcd 只会出现在 1∼⌊n3⌋1\sim\lfloor\frac{n}{3}\rfloor1∼⌊3n​⌋ 这个区间内。也就是说如果我把 1∼⌊n3⌋1\sim\lfloor\frac{n}{3}\rfloor1∼⌊3n​⌋ 内的数全部选上，那么无论 ⌊n3⌋+1∼n\lfloor\frac{n}{3}\rfloor+1\sim n⌊3n​⌋+1∼n 中的任意一个数选或者不选都是成立的。

也就是说我们已经证明了数组的长度肯定是不超过 32log⁡2n\frac{3}{2}\log_2n23​log2​n 的，而且实际值会比这个更小一些。实际上取到 404040 肯定是没有问题的。

我们再思考一下 Laikan 序下的序列怎样递推出来。

我们按照定义，尽可能地不增加大的数。但是毕竟是递推到下一个，所以说最优策略就是将当前序列的 mex⁡\operatorname{mex}mex 加入到序列中，并将 <mex⁡<\operatorname{mex}<mex 的数全部删去，最后枚举元素，将 gcd⁡\gcdgcd 给重新加回来。

这样做显然时间复杂度是不能接受的，不过你会分段打表，于是这道题就做完了。

打表代码如下：

CPP 
#include<bits/stdc++.h>
using namespace std;
#define int long long
int ans,lm,gc[42][42];
inline void nxt(int& v){
	v++;
	for(int i=39;i>=0;--i)
		for(int j=i-1;j>=0;--j)
			if((v>>i&1)&&(v>>j&1))
				v|=1ll<<gc[i+1][j+1]-1;
}
signed main() {
	freopen("chart.txt","w",stdout); 
	ios::sync_with_stdio(0); cin.tie(0);
	for(int i=1;i<=40;++i)
		for(int j=1;j<=40;++j)
			gc[i][j]=__gcd(i,j);
	for(int i=0;i<=1.5e9;++i){
		if((i&1048575)==0) cout<<ans<<",";
		nxt(ans);
	}
}

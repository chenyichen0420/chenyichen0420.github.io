思路分析

看到构造题之后，一个较为常见的想法是先看看数据范围，有没有什么相近的东西。

这里，我们注意到次数实际上只有团子个数的 $5$ 倍左右，结合范围一看，猜测大概率是 $\log m$ 或者 $\sqrt m$ 之类的。而构造题中 $\sqrt m$ 的时间复杂度属实不常见，因此优先考虑 $\log$。

既然是 $\log$，而且不带有任何余量，那大概率只能二分或者倍增了。可是具体怎么去分治呢？

分治最重要的一个问题就是我该以什么样的标准，决定他到底该往那一边走。在这里，我们不难想到去分治的决定一个团子串到哪一串上。而由于分治本身的相似性以及团子本身的不可区分性，或许让相同颜色的团子从左到右依次去占据签子是一个明智的选择。

于是，决定团子往左还是右的标准就是左边还有没有空的签子，或者右边是不是全空的（对于这种颜色的团子而言）。

感觉上第一点是难以判断的，因为我们没有办法快速的将相同颜色的团子区分出来。那我们考虑从第二点入手。

对于 $[r,n]$，我们将当前没有的决策的团子和已经确定在这个区间内的团子放到一起，并将当前正在决策的团子踢出去。如果发现可以填满，那就说明这个颜色的团子还没有填到 $r-1$。

上述的检验流程只需要一次查询就可以完成，因此总的查询次数就是 $nm\log m$ 的。

思路比较自然，瞎实现的代码如下：

CPP 
#include<bits/stdc++.h>
using namespace std;
extern int Query(const vector<int> &x);
extern void Answer(const vector<int> &a);
int n,m;
vector<int>ans[28];
vector<int>lft,tmp;
inline bool check(int p){
    tmp=lft;
    for(int i=m;i>p;i--)
        for(int j:ans[i]) tmp.emplace_back(j);
    return Query(tmp)>=m-p;
}
void Solve(int N, int M){
    n=N,m=M;
    for(int i=1;i<=N*M;++i) lft.emplace_back(i);
    reverse(lft.begin(),lft.end());
    for(int i=1;i<=N*M;++i){
        lft.pop_back();
        int l=1,r=M,mid;
        while(l!=r){
            mid=l+r>>1;
            if(check(mid)) r=mid;
            else l=mid+1;
        }
        ans[l].emplace_back(i);
    }
    for(int i=1;i<=M;++i) Answer(ans[i]);
}
